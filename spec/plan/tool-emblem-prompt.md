# tool 纹章生成提示词(喂模型用)

> 复制下面 `===` 之间整段给模型(GPT/Claude 皆可)。它会逐个 tool 产出「纹章解读 + WGSL 实现」。
> 想只生成某几个 tool,就在末尾「任务」里替换清单。

```
你是一名同时精通 **生成式 shader(SDF/WGSL)** 与 **奥运纹章设计** 的视觉工程师。
任务:为代码 agent「opencode」的内置 tool 重做一套图标。第一版太弱(直白小图标),
现在要重设计为 **极客 · 抽象 · 运动** 的动态纹章,母语借 **巴黎 2024 奥运纹章(coat of arms)**。

======================== 一、先吃透巴黎 2024 设计思路 ========================
巴黎 2024 废弃传统象形图,改为「纹章 / blason」——给每个项目一枚徽章,代表一个
「家族」的荣誉与归属,而非「一个人在做动作」。可复用的合成法是三层:
  1) 器械/动作:项目的标志性装备或动作;
  2) 场域:场地/赛道的独特特征(网、跑道、靶环…);
  3) 对称轴:把以上用 **旋转/镜像对称** 绕中心熔成一枚徽章。
气质来源:Art Deco 的强轴对称与放射/阶梯、1920s 立体主义+奥费主义的几何分面与
同心节奏、法式规则花园的对称。全程 **抽象、无具象人物**。
已知代价:为美感牺牲了「一眼看懂」。→ 你必须 **补回可读性**:靠「动势」这条轴
(动作即动词),并保证场域+器械两层都在。

======================== 二、我们的四要素配方(每枚必须四层都有) ========================
  · 场域 Field      = 该 tool 的「赛场」,但换成 **计算机母题**(终端边框、文件树、
                      网络经纬、diff 视图、符号图…)。手法:rectSDF 框 / fract 网格 / 跑道环。
  · 器械 Instrument = 该 tool 的「装备」,geek 符号系:提示符 > 、光标块、花括号 {} 、
                      尖括号 <> 、正则透镜、通配符 * 、± 、钥匙、节点…。手法:circle/tri/cross/rays SDF。
  · 对称轴 Axis     = 至少一条对称轴(竖镜最稳:st.x = abs(st.x-0.5)+0.5;
                      进阶 N 折旋转对称:对 rotate(st, k*TAU/N) 叠加)。这是「徽章感」的来源。
  · 动势 Motion     = 把「轨迹」**真的动起来**(这是我们独有、也是可读性补偿)。用 time 驱动:
                      sweep(扫掠)/orbit(环绕)/ping(扩散)/draw-on(描出)/breathe(S=1+sin(t)/4)。
立体主义/奥费主义落到 1-bit:用 **重叠几何面 + 同心弧/环的节奏 + 放射轴** 表达,别画写实物件。

======================== 三、shader 约束(必须遵守) ========================
  · 语言:WGSL。坐标 st ∈ [0,1]^2(盒内),时间 t=time。单盒 = 一枚纹章满格。
  · 只用下列 SDF 工具箱(标准数学,假定已在 base/sdf.wgsl):
      sstep, stroke(x,s,w), circleSDF(st), fillf(x,size), rectSDF(st,s2),
      crossSDF(st,s), flipf(v,pct), triSDF(st), rotate(st,a)->vec2, raysSDF(st,n),
      scale2(st,s2), seg(p,a,b), offc(st,cx,cy)->vec2, line2(p,a,b,w)。
      需要新 helper 时,只能用上面这些组合,不得引入 noise / 多 pass / raymarch(护栏:便宜片元)。
  · 输出:函数 `fn icon(st: vec2<f32>, id: i32, t: f32) -> f32` 里新增一个 `case <id>: { ... }`,
      返回 1-bit 亮度(0..1);所有图标 **全 dynamic=true**(都要用 t)。
  · 配色由上层 params 决定(青/靛 或 1-bit),case 内只算亮度,别写死颜色。
  · 对称要在坐标层做(折叠 st),不要靠手动镜像复制代码。

======================== 四、tool 清单与锚点(场域/器械/动势 提示,可优化) ========================
  shell(id50)   场域:终端边框  器械:> + 光标块        动势:caret 闪、回显流
  read(51)      场域:文件缓冲  器械:扫描头            动势:扫描线纵扫
  write(52→注)  场域:空缓冲    器械:笔尖/光标          动势:caret 推进、留迹
  edit(53)      场域:文本块    器械:选区/替换标        动势:高亮滑动+反相(flipf)
  glob(54)      场域:文件树星座 器械:通配符 *          动势:星号旋+命中波纹
  grep(55)      场域:文本行阵  器械:正则透镜          动势:命中行逐行点亮
  task(56)      场域:进程空间  器械:派生节点          动势:子节点环绕/分形
  webfetch(57)  场域:地球经纬网 器械:URL 锚/箭         动势:数据包向心
  websearch(58) 场域:网络场    器械:查询射线          动势:声呐离心
  todowrite(59) 场域:清单栏    器械:复选框            动势:勾选推进(draw-on)
  skill(60)     场域:能力槽    器械:钥匙/模块          动势:插入解锁微旋
  apply_patch(61) 场域:diff 视图 器械:± hunk          动势:增删行流入,±交替
  question(62)  场域:对话框    器械:?/分支            动势:等待脉冲(breathe)
  lsp(66)       场域:符号/AST 图 器械:{} 探针          动势:诊断点跳
  plan-enter(63)/plan-exit(64) 场域:蓝图栅格 器械:节点路由 动势:折线描出→播放三角
  invalid(65)   场域:空集 ∅    器械:禁止环            动势:glitch 抖
  (注:id 与 plan16 §2.5c 注册表对齐;edit/write 实际 id 以注册表为准。)

======================== 五、风格锚(照此质感产出)—— 一个完整范例 ========================
// shell —— 纹章解读:
//   场域=终端「门框」(上下两道横楣 + 竖镜对称的侧柱,Art Deco 阶梯感);
//   器械=居中提示符 > (两臂在轴上交汇)+ 下方光标块;
//   轴=竖镜(st.x 折叠),门框天然左右对称;
//   动势=光标块 blink + 一道回显扫线沿底楣 sweep。
case 50: {
    var p = st;
    p.x = abs(p.x - 0.5) + 0.5;                 // 竖镜对称(徽章轴)
    // 场域:门框横楣 + 侧柱(阶梯放射)
    var c = stroke(p.y, 0.82, 0.05) + stroke(p.y, 0.18, 0.05);
    c += stroke(p.x, 0.84, 0.05) * step(0.16, p.y) * step(p.y, 0.84);
    // 器械:> 提示符(轴上交汇的下臂;镜像后自动成 > 形)
    let arm = 0.5 + (p.y - p.x) * 0.5;
    c += stroke(arm, 0.42, 0.055) * step(0.30, p.x) * step(p.x, 0.66)
                                  * step(0.30, p.y) * step(p.y, 0.62);
    // 器械:光标块,闪
    c += fillf(rectSDF(offc(st, 0.5, 0.34), vec2<f32>(0.07, 0.05)), 0.5)
         * step(0.5, fract(t * 1.6));
    // 动势:回显扫线沿底楣
    c += stroke(st.x, fract(t * 0.5), 0.02) * step(0.20, st.y) * step(st.y, 0.26);
    color = clamp(c, 0.0, 1.0);
}

======================== 六、产出格式(逐 tool) ========================
对清单里的每个 tool,输出两块:
  【纹章解读】4 行:场域 / 器械 / 对称轴 / 动势,各一句(说清用了哪些 SDF 与运动习语)。
  【WGSL】一个可编译的 `case <id>: { ... }`,只用第三节工具箱,坐标层做对称,
          末行 `color = clamp(c, 0.0, 1.0);`。
全部完成后,再给一段「整体评述」:这套相比第一版,如何体现极客/抽象/运动 + 巴黎纹章三层法。

skill, apply_patch, question, plan-enter, plan-exit, invalid。
逐个给【纹章解读】+【WGSL case】,严格遵守上面六节。
```

## 用法说明

1. 上面整段代码块复制给模型 → 得到 16 枚纹章的解读 + WGSL case。
2. 把生成的 `case` 贴进 `tool-icons.wgsl` 的 `icon()` switch(替换或新增 id),工具箱已就绪。
3. GLSL contact-sheet 同理:把 WGSL 语法回译成 GLSL(`fillf→fill`、`let→float`、`f32(i)→float(i)`、`atan2→atan`)即可在 shadertoy 验收。
4. 不满意某枚 → 单独把该 tool 的【纹章解读】+ 一句修改意见再喂一次,迭代成本低。

> 提示词本身只编码「方法论 + 约束 + 一个范例」,不锁死具体造型——模型有空间发挥,但被巴黎三层法 + shader 护栏框住,产出不会跑偏。
